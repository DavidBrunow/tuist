# Showroom

As a side effect of making the description of a project easier, describing projects to others is easier too. This page contains examples of project configurations: *the folders structure and the manifest files.*

> If your project uses Tuist and you would like to share your configuration with others don't hesitate to open a PR and include your example here. Don't forget to add a reference to your company or GitHub username.

## Setup 1 - Microfeatures

Microfeatures is a proposal for organizing modular projects that suggests placing features in modules _(frameworks or libraries)_ that are distributed horizontally. As a result, changes in core modules don't invalidate a large dependency graph causing builds to be significantly slow.
